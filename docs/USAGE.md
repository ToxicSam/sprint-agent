## 🚀 快速入门

### 首次登录

打开 Sprint Agent v2.0 后，您将看到登录页面：

1. 🔐 **选择角色**：
   - **Scrum Master (Scrum主管)** 🏆 — 完整权限，可管理迭代、成员、设置
   - **Developer (开发人员)** 👨‍💻 — 受限权限，可操作任务、参与站会、查看看板

2. 📂 **数据导入选项**（可选）：
   - 如果您已有 Sprint Agent 的 JSON 数据文件，可点击「📥 导入数据」直接加载
   - 支持拖拽上传，系统自动验证数据格式

3. 🔄 **自动跳转**：
   - 若系统中已存在迭代，自动跳转到看板仪表盘
   - 若无迭代数据，自动跳转到迭代规划页面

> 💡 **小贴士**: Scrum Master 建议首次使用时先完成「➕ 创建新迭代」的流程

## 📊 看板仪表盘

仪表盘是 Sprint Agent 的核心页面，采用 **4 列看板布局** 管理任务。

### 🎯 页面结构

顶部 → **迭代信息栏** → **筛选工具栏** → **4 列看板** → **Agent 面板（右侧停靠）**

### 📋 看板列说明

| 列 | 标识 | 说明 |
|---|---|---|
| 🟡 未开始 | `todo` | 已规划但尚未开始的任务 |
| 🔵 进行中 | `progress` | 正在积极处理的任务 |
| 🟢 已完成 | `done` | 已验收完成的任务 |
| 🔴 已暂停 | `paused` | 因依赖或阻塞暂停的任务 |

### 🖱️ 拖拽操作

- **拖拽卡片**：按住任务卡片任意位置，拖动到目标列
- **视觉反馈**：拖拽时卡片半透明，目标列高亮显示
- **自动排序**：释放后卡片自动插入到目标列适当位置
- **状态同步**：拖拽后任务状态自动更新，统计数字实时变化

> 💡 拖拽使用 `@dnd-kit` 库，支持平滑动画和触控设备

### 📝 任务卡片

每张卡片包含以下信息：

- 🏷️ **优先级标签**：
  - 🔴 紧急 — 红色标签
  - 🟠 高 — 橙色标签
  - 🟡 中 — 黄色标签
  - 🔵 低 — 蓝色标签
- 📝 **任务标题** — 清晰描述工作内容
- 👤 **负责人头像** — 显示圆形头像，悬停显示姓名
- 🔢 **故事点** — 右下角显示估算点数
- ⬌ **拖拽手柄** — 左侧竖条，提示可拖拽

### ➕ 行内创建任务

看板的 **核心交互创新** — 无需弹窗，直接在列内创建：

1. 每列底部有一个「幽灵卡片」（半透明显示）
2. 🖱️ **点击幽灵卡片** → 展开为输入表单
3. ⌨️ **输入任务标题**，按 `Tab` 切换字段
4. ✅ **按 Enter** — 立即创建任务，卡片滑入列中
5. ❌ **按 Esc** — 取消创建，恢复幽灵卡片

> ⚡ **效率对比**: 传统方式（打开弹窗→填写5个字段→确认）需 15+ 秒 | Sprint Agent 行内创建仅需 3 秒

### 🔍 筛选工具栏

| 功能 | 操作 |
|---|---|
| 🔎 搜索 | 输入关键词，实时过滤任务标题和描述 |
| 🏷️ 状态筛选 | 下拉选择，只看特定状态的任务 |
| 👤 负责人筛选 | 选择团队成员，查看其负责的任务 |
| ❌ 清除筛选 | 点击「重置」按钮恢复全部显示 |

### 📈 迭代统计栏

迭代信息栏显示当前 Sprint 的关键指标：

```
📌 Sprint 14 — "支付模块优化" | 2024.01.15 - 2024.01.29
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│   32    │   18    │    8    │    4    │    2    │
│  总计   │ 已完成  │ 进行中  │  已暂停  │  阻塞   │
└─────────┴─────────┴─────────┴─────────┴─────────┘
```

## 🗓️ 迭代规划

规划页面采用 **双栏布局**：左侧 Sprint 设置（60%）+ 右侧团队成员（40%）。

### 📝 创建 Sprint

1. **📛 名称输入**：
   - 系统自动生成建议名称：`Sprint DD.MM`（如 `Sprint 15.01`）
   - 可手动修改，支持任意命名

2. **🎯 目标描述**：
   - 多行文本框，描述本次迭代的核心目标
   - 例：「完成支付流程重构，提升 20% 处理性能」

3. **📅 日期范围**：
   - 点击选择开始日期和结束日期
   - 联动模板预设，自动计算周期

4. **🔘 状态切换**：
   - 激活 / 未激活
   - 一个团队同时只能有一个「激活」的 Sprint

### 🎛️ 模板预设

| 模板 | 周期 | 适用场景 |
|---|---|---|
| ⚡ 快速 1 周 | 7 天 | 紧急修复、小规模迭代 |
| 📋 标准 2 周 | 14 天 | 常规开发迭代（推荐） |
| 🔧 扩展 3 周 | 21 天 | 大型功能、架构升级 |

> 点击模板按钮后，日期范围自动填充，名称自动建议

### 👥 团队成员管理

#### 成员表格

| 字段 | 说明 |
|---|---|
| 👤 姓名 | 团队成员姓名 |
| 🏷️ 角色 | 角色标签（见下） |
| ⚡ 容量 | 每日可用工时（小时） |

#### 角色标签颜色

| 角色 | 缩写 | 颜色 |
|---|---|---|
| 🟣 Scrum Master | SM | 紫色 |
| 🔵 开发人员 | Dev | 蓝色 |
| 🟠 测试人员 | QA | 琥珀色 |
| 🟢 产品负责人 | PO | 绿色 |

#### 行内添加成员

1. 表格底部有一行空行
2. 按 `Tab` 键 → 自动创建新行
3. 输入姓名、选择角色、填写容量
4. Enter 保存，自动进入下一行

> 💡 **连续输入流**：支持不间断添加多个成员，无需点击任何按钮

#### ✏️ 行内编辑

- **双击单元格** 或 点击 ✏️ 编辑图标 → 进入编辑模式
- 修改内容后按 **Enter** 保存
- 按 **Esc** 取消编辑

#### 🗑️ 删除成员

- 点击成员行的 🗑️ 删除图标
- 弹出确认对话框（防止误操作）
- 确认后移除成员，容量自动重新计算

#### 📊 容量自动计算

系统自动汇总所有成员的每日容量：
```
团队总容量 = Σ(成员容量 × 迭代工作日)
例：5人 × 6小时/天 × 10天 = 300 小时
```

## ✅ 每日站会

> ⭐ **v2.0 最大 UX 改进** — Excel 风格的批量编辑表格

### 🎯 设计理念

**传统站会工具的问题**：
- 多步骤向导，每人需要 30+ 秒
- 弹窗表单打断流程
- 逐人填写效率低

**Sprint Agent 的解决方案**：
- 🖥️ **单屏操作**，无需向导、无需弹窗
- ⌨️ **键盘优先**，Tab 导航、Enter 提交
- ⚡ **批量编辑**，5 秒完成一个人的站会汇报

### 📋 页面布局

单行表头 + 多行数据，类似 Excel 表格：

| 成员 👤 | 昨日计划 📋(只读) | 实际完成 ✅ | 今日计划 📅 | 阻塞项 🚧 | 工时 ⏰ |
|---|---|---|---|---|---|
| 张三 | 完成登录接口 | 完成登录接口 | 开发用户中心 | 无 | 6 |
| 李四 | 编写测试用例 | 编写测试用例 | 执行接口测试 | API文档缺失 | 5 |
| 王五 | 设计数据库 | 设计数据库 | 编写迁移脚本 | 无 | 7 |

### ⌨️ 键盘导航（核心！）

| 快捷键 | 动作 |
|---|---|
| `Tab` | 向右移动到下一个单元格 |
| `Shift + Tab` | 向左移动到上一个单元格 |
| `Enter` | 向下移动到下一行（同列） |
| `Shift + Enter` | 向上移动到上一行（同列） |
| `Ctrl + Enter` | **提交所有行**（站会完成） |

> 💡 完全模拟 Excel 的操作习惯，无需学习成本

### 🔘 快捷按钮

| 按钮 | 功能 | 说明 |
|---|---|---|
| 📋 填充昨日 | 自动将「昨日计划」填入「实际完成」 | 计划顺利时一键填充 |
| 📄 复制昨日 | 复制整列昨日计划 | 用于批量参考 |
| 🧹 清空全部 | 清空所有输入 | 带二次确认，防止误操作 |

### 💾 自动保存

- 所有输入 **自动保存到 localStorage**
- 意外刷新页面后数据不丢失
- 提交成功后自动清除草稿

### 📊 进度提示

页面顶部显示进度徽标：
```
📝 已填写: 5/8 人
```
全部填写后徽标变为 ✅ 绿色

### ⏰ 工时验证

- 输入范围：0 - 24 小时
- 超出范围自动标红提示
- 支持小数（如 6.5 小时）

### 🔄 站会完整流程

```
1. 打开站会页面 → 自动加载昨日计划（只读列）
2. 点击第一个成员的「实际完成」单元格
3. Tab 在单元格间移动，逐人填写
4. 计划顺利 → 点击「📋 填充昨日」一键填充
5. 全部填写 → Ctrl + Enter 提交
6. 系统自动生成站会报告
```

## 🔄 迭代回顾

### 📊 四维度评分

4 张评分卡片，每张 1-5 星：

| 维度 | 说明 | 颜色 |
|---|---|---|
| ⚡ 速度 | 迭代完成的故事点与计划对比 | 🟢 高分=超额完成 |
| 🎯 质量 | 代码质量、缺陷率、测试覆盖 | 🟢 高分=质量高 |
| 🤝 协作 | 团队沟通、知识分享、互助 | 🟢 高分=协作好 |
| 📋 流程 | 会议效率、工具使用、流程顺畅 | 🟢 高分=流程优 |

#### 评分方式

- 🖱️ **鼠标点击** 星星评分
- ⌨️ **按数字键 1-5** 快速评分
- 🎨 **颜色编码**：
  - 🔴 1-2 分：红色，需要重点关注
  - 🟡 3 分：黄色，有待改进
  - 🟢 4-5 分：绿色，表现优秀

### 📝 反馈标签页

3 个标签页收集团队反馈：

| 标签 | 说明 | 示例 |
|---|---|---|
| 👍 做得好的 | 本迭代的成功经验 | "每日站会效率很高" |
| 👎 待改进的 | 发现的问题和不足 | "代码审查时间不够" |
| ✅ 行动项 | 具体的改进措施 | "增加 30 分钟代码审查时间" |

#### 添加反馈项

1. 选择对应标签页
2. 在底部输入框输入内容
3. 按 **Enter** 立即添加
4. 新项滑入列表，无需刷新

#### 👍 投票功能

- 每条反馈右侧有 👍 点赞按钮
- 点击切换点赞/取消
- 投票数用于排序和生成报告

### 🎨 模板选择器

| 模板 | 分类维度 | 适用场景 |
|---|---|---|
| 😡😢😊 Mad/Sad/Glad | 情绪维度 | 快速情绪回顾 |
| 🍀 4Ls | Liked/Learned/Lacked/Longed for | 学习型团队 |
| ⭐ Starfish | 保持/减少/增加/开始/停止 | 深度改进 |
| 🚦 Start/Stop/Continue | 行动导向 | 简洁高效回顾 |

> 切换模板时，标签页自动重新命名，已有内容保留

### 📄 生成总结报告

评分完成后：

1. 点击「📄 生成报告」按钮
2. 系统自动汇总：
   - 📊 四维度平均分及雷达图
   - 📋 各标签页反馈汇总
   - 🏆 投票最多的 Top 反馈
   - ✅ 行动项清单
3. 支持 **导出 Markdown** 文件

## 🤖 智能助手面板

Agent 面板固定在页面 **右侧（320px 宽度）**，提供 AI 辅助功能。

### 📐 面板结构

```
┌─────────────────────────────┐
│  🤖 Sprint Agent      [⬌]  │  ← 标题栏 + 折叠按钮
├─────────────────────────────┤
│  [📝 新任务] [📊 报告]     │  ← 快捷操作按钮
├─────────────────────────────┤
│                             │
│  🤖: 有什么可以帮您的？     │  ← 消息气泡区
│                             │
│  👤: 帮我分析当前进度       │
│                             │
│  🤖: 当前 Sprint 已完成     │
│     56%，预计按时完成...    │
│                             │
├─────────────────────────────┤
│  > /plan 创建新迭代          │  ← 输入框
└─────────────────────────────┘
```

### 🎭 快捷操作按钮

面板顶部提供常用功能一键触发：

| 按钮 | 功能 |
|---|---|
| 📝 新任务 | 在当前看板创建新任务 |
| 📊 报告 | 生成当前 Sprint 进度报告 |
| 🔄 站会 | 打开站会页面 |
| ⚙️ 设置 | 打开设置页面 |

### ⌨️ Slash 命令

在输入框中输入 `/` 触发命令列表：

| 命令 | 功能 | 示例 |
|---|---|---|
| `/plan` | 创建新迭代 | `/plan 支付模块优化 1月15日-1月29日` |
| `/standup` | 快速站会 | `/standup 查看今日站会状态` |
| `/retro` | 生成本迭代回顾报告 | `/retro 分析速度和质量维度` |
| `/help` | 显示帮助信息 | `/help` |

### 🧠 上下文感知

Agent 具备智能上下文感知能力：

- 📍 **页面感知**：知道您当前在看板/规划/站会/回顾页面
- 📝 **任务感知**：知道您当前选中的任务
- 👤 **角色感知**：根据您的角色（SM/Dev）调整回复内容
- 📊 **数据感知**：实时访问当前 Sprint 数据

### ✏️ 输入状态

- 输入时显示 **闪烁动画** 的打字指示器
- 三个跳动的圆点表示 Agent 正在思考
- 响应通常在 1-2 秒内返回

### ⬌ 折叠/展开

- 点击标题栏的 ⬌ 按钮折叠面板
- 折叠后显示为圆形 🤖 图标（右下角）
- 点击图标重新展开

> 💡 **快捷键**: 按 `A` 键快速切换 Agent 面板显示/隐藏

| `Enter` | 保存编辑 |
| `Esc` | 取消编辑 |

## 📥 数据导入指南

### 📋 支持的格式

Sprint Agent 支持导入 **JSON** 格式的数据文件。

### 📁 JSON 文件结构

```json
{
  "version": "2.0",
  "sprints": [
    {
      "id": "sprint-001",
      "name": "Sprint 01.15",
      "goal": "完成支付模块重构",
      "startDate": "2024-01-15",
      "endDate": "2024-01-29",
      "status": "active",
      "tasks": [
        {
          "id": "task-001",
          "title": "设计支付接口",
          "status": "done",
          "priority": "high",
          "assignee": "张三",
          "storyPoints": 5
        }
      ]
    }
  ],
  "members": [
    {
      "name": "张三",
      "role": "Dev",
      "capacity": 6
    }
  ]
}
```

### 🔄 导入步骤

1. 打开 **设置 → 数据导入/导出**
2. 将 JSON 文件 **拖拽** 到上传区域
3. 系统自动进行 **Schema 验证**
4. 📋 **预览数据**：
   - 显示检测到的迭代数量
   - 显示成员数量
   - 显示任务总数
   - 标记潜在问题（红色警告）
5. 点击「✅ 确认导入」应用数据
6. 导入成功后自动跳转到看板

### ✅ Schema 验证规则

| 检查项 | 说明 |
|---|---|
| 版本兼容 | 必须 >= 2.0 |
| 必填字段 | name, status, tasks |
| 日期格式 | ISO 8601 (YYYY-MM-DD) |
| 状态枚举 | todo / progress / done / paused |
| 优先级枚举 | urgent / high / medium / low |
| 角色枚举 | SM / Dev / QA / PO |

### ⚠️ 常见导入错误

| 错误信息 | 原因 | 解决方法 |
|---|---|---|
| `版本不兼容` | 文件版本 < 2.0 | 升级数据或重新导出 |
| `缺少必填字段` | JSON 缺少关键字段 | 检查并补充字段 |
| `日期格式错误` | 日期不是 YYYY-MM-DD | 修正日期格式 |
| `状态值无效` | status 不在允许列表中 | 修正为有效值 |
| `故事点非数字` | storyPoints 不是数字 | 改为数字类型 |

## ❓ 常见问题

### 🚀 入门相关

**Q1: 我是第一次使用，应该选什么角色？**
> 如果您是团队负责人或 Scrum Master，选择 🏆 Scrum Master（完整权限）。如果是普通开发人员，选择 👨‍💻 Developer。

**Q2: 没有现有数据，能试用吗？**
> 可以！系统提供默认示例数据，在设置中点击「🔄 重置为默认数据」即可加载演示数据。

**Q3: 如何切换角色？**
> 需要退出重新登录。点击右上角头像 → 退出 → 重新选择角色。

### 📊 看板相关

**Q4: 任务卡片可以拖拽到任意列吗？**
> 是的！所有列之间都支持自由拖拽，状态会自动同步更新。

**Q5: 如何快速创建多个任务？**
> 在看板页面连续按 `N` → 输入标题 → `Enter` → 自动创建并准备下一张卡片。无需鼠标操作。

**Q6: 任务的故事点可以修改吗？**
> 可以。按 `E` 编辑任务，或双击任务卡片进入编辑模式。

**Q7: 看板支持移动端吗？**
> 支持！在手机上横向滑动即可查看所有列。触控设备同样支持拖拽。

**Q8: 筛选后拖拽任务会改变筛选结果吗？**
> 不会。筛选仅影响显示，不改变实际数据。清除筛选后所有任务恢复显示。

### ✅ 站会相关

**Q9: 站会页面的「昨日计划」列为什么是只读的？**
> 这是系统根据前一天的「今日计划」自动生成的，确保数据一致性。

**Q10: 如果今天有人请假，站会怎么填？**
> 在「实际完成」中填写「请假」或「休假」，工时为 0 即可。

**Q11: 站会数据提交后能修改吗？**
> 当天可以修改。进入站会页面会自动加载当天的草稿，修改后重新提交即可。

**Q12: 「填充昨日」按钮会覆盖已填写的内容吗？**
> 不会。只填充为空的单元格，已有内容不会被覆盖。

### 🔄 回顾相关

**Q13: 回顾评分可以修改吗？**
> 可以。在生成最终报告前，随时可以重新点击星星修改评分。

**Q14: 不同模板之间切换，已填写的反馈会丢失吗？**
> 不会。反馈内容保留，只是标签页的分类名称会变化。

**Q15: 回顾报告可以分享给不在系统内的人吗？**
> 可以。导出为 Markdown 文件后，可以通过邮件、即时通讯工具分享。

### 🤖 Agent 相关

**Q16: Agent 面板会占用太多屏幕空间吗？**
> 默认宽度 320px，可以折叠为圆形图标。按 `A` 键快速切换。

**Q17: Agent 能理解中文指令吗？**
> 是的！Agent 完全支持中文对话和指令。

**Q18: Agent 的数据从哪里来？**
> Agent 实时访问您当前页面的数据（任务、成员、Sprint 信息），所有分析基于真实数据。

### ⚙️ 设置相关

**Q19: 如何备份所有数据？**
> 设置 → 数据导入/导出 → 选择 JSON 格式 → 导出。建议定期备份。

**Q20: 导入数据会覆盖现有数据吗？**
> 是的。导入前系统会提示确认，建议先导出备份。

**Q21: 主题设置为「跟随系统」会怎样？**
> 系统会自动检测您操作系统的主题设置（浅色/深色模式）并同步切换。

**Q22: 可以自定义 Agent 的人格吗？**
> 可以。设置 → Agent → 选择专业/友好/直接三种风格之一。

### 🔧 技术相关

**Q23: 数据存储在哪里？**
> 所有数据存储在浏览器本地（localStorage）。清除浏览器数据会导致丢失。

**Q24: 多人能同时使用吗？**
> 当前版本为单机版，数据不自动同步。建议由 Scrum Master 统一管理数据。

**Q25: 支持哪些浏览器？**
> 推荐 Chrome、Edge、Firefox、Safari 最新版本。IE 不支持。

## 🔧 故障排除

### 🐛 常见问题解决

#### 问题 1: 看板页面空白，不显示任务

| 可能原因 | 解决方法 |
|---|---|
| 无激活的 Sprint | 前往 🗓️ 规划页面创建一个 Sprint |
| 数据未加载 | 刷新页面（F5），检查控制台错误 |
| 筛选条件导致 | 点击「重置」清除所有筛选条件 |

#### 问题 2: 站会页面无法提交

| 可能原因 | 解决方法 |
|---|---|
| 工时超出范围 | 确保工时输入为 0-24 之间的数字 |
| 有必填项为空 | Tab 检查每个单元格，确保无遗漏 |
| 浏览器缓存问题 | 按 Ctrl + F5 强制刷新 |

#### 问题 3: 数据导入失败

| 错误提示 | 解决方法 |
|---|---|
| `版本不兼容` | 确认 JSON 文件 version >= 2.0 |
| `Schema 验证失败` | 对照上面的 JSON 结构检查字段 |
| `日期格式错误` | 将日期改为 YYYY-MM-DD 格式 |
| `未知角色值` | 角色只能是 SM/Dev/QA/PO 之一 |

#### 问题 4: Agent 面板不响应

| 可能原因 | 解决方法 |
|---|---|
| 面板已折叠 | 按 `A` 键或点击右下角 🤖 图标 |
| 页面未完全加载 | 等待页面加载完成后再使用 |
| 浏览器扩展冲突 | 尝试在无痕模式下打开 |

#### 问题 5: 键盘快捷键无效

| 可能原因 | 解决方法 |
|---|---|
| 焦点在输入框内 | 先按 Esc 退出输入模式 |
| 输入法冲突 | 切换到英文输入法再试 |
| 快捷键被浏览器占用 | 尝试 Ctrl + 快捷键组合 |

#### 问题 6: 拖拽功能不工作

| 可能原因 | 解决方法 |
|---|---|
| 使用了触控设备 | 确保长按 200ms 后再拖动 |
| 浏览器不支持 | 升级浏览器到最新版本 |
| 页面缩放问题 | 按 Ctrl + 0 恢复 100% 缩放 |

### 🆘 获取帮助

如果以上方法无法解决问题：

1. 📸 **截图** 问题页面和浏览器控制台（F12 → Console）
2. 📋 **导出数据** 备份当前状态
3. 🔄 **重置数据** 作为临时解决方案
4. 📝 **记录复现步骤** 便于排查

### 🔄 数据恢复

如果不小心删除了重要数据：

1. 检查是否有 **JSON 导出备份**
2. 尝试使用浏览器的 **撤销操作**（Ctrl + Z）
3. 如无备份，需要重新手动输入

> ⚠️ **重要提示**: 建议每个 Sprint 结束后导出 JSON 备份！数据仅存在浏览器本地，清除缓存会导致丢失。
