# 🎨 Sprint Agent v2.0 UX 设计原则文档

> **版本**: v2.0 | **定位**: 产品设计理念与体验优化方法论 | **阅读对象**: 产品、设计、研发团队

## 1. 🧠 设计哲学：「处处快操作」

Sprint Agent v2.0 的核心设计理念 —— **「Quick Actions Everywhere / 处处快操作」**。这一理念贯穿于每一个页面、每一个交互、每一个快捷键设计之中。

### 三大设计铁律

| 原则 | 说明 | 实践示例 |
|---|---|---|
| ⚡ **≤3 步完成** | 任何操作应在 3 步之内完成 | 创建任务：点击→输入→Enter |
| ✏️ **行内优先** | 优先使用行内编辑，避免弹窗 | 看板直接创建任务，无需弹窗 |
| ⌨️ **键盘优先** | 键盘操作效率高于鼠标 | Tab/Enter 导航，快捷键全局覆盖 |

### 效率设计公式

```
用户效率 = 操作步骤数 × 认知负载 × 切换成本

传统工具: 5 步 × 高认知 × 高切换成本 = 低效
Sprint Agent: 2 步 × 低认知 × 无切换成本 = 高效
```

### 核心理念解读

**「处处快操作」** 不是简单的功能堆砌，而是对敏捷团队工作流的深度理解：

- 🏃 **敏捷团队时间宝贵** — 站会只有 15 分钟，回顾只有 1 小时
- 🧠 **减少上下文切换** — 每打开一个弹窗，认知负载增加 40%
- ⚡ **操作即思考** — 工具不应打断思路，而应跟随思维流动
- 📱 **一次到位** — 所有操作在当前上下文完成，无需跳转

## 2. ✅ 站会 UX 案例研究：从 30 秒到 5 秒的蜕变

### 🎯 问题分析

**传统站会工具的操作流程**（典型多步向导模式）：

```
1. 点击「添加站会记录」→ 打开弹窗（等待 0.5s）
2. 选择成员（下拉搜索）→ 填写昨日完成
3. 切换到「今日计划」标签 → 输入今日计划
4. 切换到「阻塞项」标签 → 输入阻塞项
5. 输入工时 → 点击保存
6. 关闭弹窗 → 重复下一个人

⏱️ 耗时: 每人 30+ 秒 | 8 人团队 = 4 分钟+
😰 问题: 弹窗打断、逐人操作、无法整体查看
```

### ✅ Sprint Agent 解决方案

**Excel 风格的批量编辑表格**：

```
1. 打开站会页面 → 自动加载昨日计划（只读）
2. 点击第一个单元格 → Tab 导航 → Enter 下一个人

⏱️ 耗时: 每人 5 秒 | 8 人团队 = 40 秒
✨ 优势: 单屏操作、键盘导航、批量提交
```

### 📊 关键改进点

| 维度 | 传统方式 | Sprint Agent | 改进倍数 |
|---|---|---|---|
| ⏱️ 每人耗时 | 30 秒 | 5 秒 | **6x** |
| 🖱️ 点击次数 | 15+ 次 | 3 次 | **5x** |
| 🧠 认知切换 | 5 次/人 | 1 次 | **5x** |
| 📱 支持并行查看 | ❌ 否 | ✅ 是 | — |
| ⌨️ 键盘操作 | ❌ 有限 | ✅ 全面 | — |

### 💡 设计洞察

**核心洞察**: 站会是一个「批量收集信息」的场景，却被设计成了「逐人录入」的交互。

- 📋 **行内编辑** 消除了弹窗的上下文切换
- ⌨️ **Tab/Enter 导航** 模拟 Excel 操作习惯，零学习成本
- 📊 **进度徽标** 提供实时反馈，站会主持人一目了然
- 💾 **自动保存** 消除意外丢失的焦虑

## 3. 📝 任务创建 UX 案例研究：从 15 秒到 3 秒

### 🎯 问题分析

**传统看板工具的任务创建**（弹窗表单模式）：

```
1. 点击「新建任务」按钮 → 打开弹窗
2. 填写标题输入框
3. 切换到优先级下拉框 → 选择优先级
4. 切换到负责人选择器 → 搜索并选择
5. 切换到故事点输入框 → 输入点数
6. 切换到描述文本框 → 输入描述
7. 点击「创建」按钮
8. 等待弹窗关闭 → 看板刷新

⏱️ 耗时: 15+ 秒 | 😰 打断看板查看上下文
```

### ✅ Sprint Agent 解决方案

**行内幽灵卡片**：

```
1. 点击列底部幽灵卡片 → 展开输入框
2. 输入标题 → 按 Enter

⏱️ 耗时: 3 秒 | ✨ 看板内直接创建，无弹窗
```

### 📊 关键改进点

| 维度 | 传统方式 | Sprint Agent | 改进倍数 |
|---|---|---|---|
| ⏱️ 耗时 | 15 秒 | 3 秒 | **5x** |
| 🖱️ 点击次数 | 8+ 次 | 2 次 | **4x** |
| 🧠 认知切换 | 4 次（字段间） | 0 次 | **∞** |
| 📍 上下文保持 | ❌ 弹窗打断 | ✅ 保持看板上下文 | — |
| ⚡ 连续创建 | ❌ 需重复打开弹窗 | ✅ Enter 后自动下一个 | — |

### 💡 设计洞察

**核心洞察**: 敏捷团队需要「快速记录想法」，而不是「填写表单」。

- 👻 **幽灵卡片** 始终以半透明状态存在，暗示可创建，降低行动门槛
- ⚡ **Enter 即创建** 将最常见的操作（创建后立刻创建下一个）优化到极致
- 🎯 **最小字段** 仅标题为必填，其他字段通过智能默认值处理
- 🔄 **连续流** 创建→Enter→创建→Enter，形成心流状态

## 4. 🔄 回顾 UX 案例研究：从繁琐到流畅

### 🎯 问题分析

**传统回顾工具的操作流程**（逐步评分模式）：

```
1. 点击「开始回顾」→ 进入第一步
2. 评分「速度」→ 选择 1-5 → 下一步
3. 评分「质量」→ 选择 1-5 → 下一步
4. 评分「协作」→ 选择 1-5 → 下一步
5. 评分「流程」→ 选择 1-5 → 下一步
6. 切换到反馈输入 → 逐条添加
7. 分别切换到 3 个标签页输入
8. 生成报告

⏱️ 耗时: 15-20 分钟 | 😰 步骤繁琐、切换频繁
```

### ✅ Sprint Agent 解决方案

**数字键快速评分 + 模板驱动**：

```
1. 直接在看板上按 1-5 数字键评分（4 个维度）
2. 切换标签页 → Enter 添加反馈
3. 一键生成报告

⏱️ 耗时: 5 分钟 | ✨ 全部在当前页面完成
```

### 📊 关键改进点

| 维度 | 传统方式 | Sprint Agent | 改进倍数 |
|---|---|---|---|
| ⏱️ 总耗时 | 15-20 分钟 | 5 分钟 | **4x** |
| 🖱️ 点击次数 | 30+ 次 | 8 次 | **4x** |
| 🔄 页面切换 | 8+ 次 | 2 次 | **4x** |
| 🎨 模板支持 | ❌ 无 | ✅ 4 种预设 | — |
| 👍 即时反馈 | ❌ 分页 | ✅ 实时显示 | — |

### 💡 设计洞察

**核心洞察**: 回顾会议的核心是「讨论」而非「操作工具」，工具应该尽可能透明。

- 🔢 **数字键评分** 消除了鼠标寻找星星的精确操作
- 🎨 **模板系统** 让团队专注于内容，而非回顾框架
- 🏷️ **行内添加反馈** 讨论中想到就输入，不中断讨论流
- 📊 **自动报告** 消除手动整理的痛苦

## 5. 🤖 Agent 集成 UX：从浮动面板到工作流伙伴

### 🎯 问题分析

**传统 AI 助手的设计问题**（浮动聊天窗口）：

```
❌ 悬浮在页面上方，遮挡内容
❌ 与当前工作上下文脱节
❌ 需要手动复制粘贴信息
❌ 只是「另一个聊天窗口」，未融入工作流
❌ 打开/关闭操作繁琐
```

### ✅ Sprint Agent 解决方案

**右侧停靠 + 上下文感知 + Slash 命令**：

```
✅ 固定 320px 宽度，不遮挡主要内容
✅ 自动感知当前页面和选中任务
✅ 快捷操作按钮直接触发功能
✅ 按 A 键一键显示/隐藏
✅ Slash 命令直接执行操作
```

### 📊 关键改进点

| 维度 | 传统 AI 助手 | Sprint Agent Agent | 改进 |
|---|---|---|---|
| 📍 位置 | 悬浮遮挡 | 右侧固定停靠 | 不遮挡 |
| 🧠 上下文 | 无感知 | 感知页面/任务/角色 | 智能 |
| ⚡ 操作 | 纯对话 | 按钮 + 命令 + 对话 | 多维 |
| 🔄 切换成本 | 高（打开→输入→等待） | 低（A 键切换） | 快速 |
| 🤝 工作流融合 | 分离 | 深度集成 | 一体 |

### 💡 设计洞察

**核心洞察**: Agent 不应该是一个「聊天工具」，而应该是「工作流的延伸」。

- 🏗️ **右侧停靠** 让 Agent 成为界面的一部分，而非覆盖层
- 🧠 **上下文感知** 让 Agent 知道用户在做什么，无需重复说明
- ⚡ **快捷按钮** 让高频操作无需输入文字
- ⌨️ **Slash 命令** 让熟练用户快速触发操作
- 🎭 **人格选择** 让不同团队找到适合自己的交互风格

## 6. ⌨️ 键盘快捷键设计原则

### 设计方法论

Sprint Agent 的快捷键设计遵循 **「肌肉记忆优先」** 原则：

| 原则 | 说明 | 示例 |
|---|---|---|
| 🎯 单键优先 | 高频操作使用单键 | `N` 新建任务，`A` 切换 Agent |
| 📍 页面隔离 | 同键在不同页面功能不同 | Enter: 看板=编辑，站会=下一行 |
| ⌨️ 组合克制 | 尽量少用组合键 | 仅保留 Ctrl+Enter 等必要组合 |
| 🧠 语义映射 | 键位与功能语义关联 | `D` = Delete，`E` = Edit |
| 📱 不冲突 | 不与浏览器默认快捷键冲突 | 避开 Ctrl+T（浏览器新建标签） |

### 快捷键设计矩阵

```
全局层:
  A = Agent 面板
  T = 主题切换
  ? = 快捷键帮助

看板层（页面激活时）:
  N = 新建任务
  1-4 = 聚焦列
  E = 编辑
  D = 删除

站会层（页面激活时）:
  Tab = 下一单元格
  Enter = 下一行（同列）
  Ctrl+Enter = 提交全部

回顾层（页面激活时）:
  1-5 = 评分
  Tab = 切换卡片
```

## 7. 📦 批量操作设计模式

### 模式定义

批量操作是 Sprint Agent v2.0 的核心 UX 模式之一，应用于多个页面：

| 页面 | 批量操作 | 实现方式 |
|---|---|---|
| ✅ 站会 | 批量填写站会记录 | Excel 表格，Ctrl+Enter 提交 |
| 📝 回顾 | 批量添加反馈 | 多标签页，Enter 连续添加 |
| 👥 规划 | 批量添加成员 | Tab 创建行，连续输入 |
| 📊 看板 | 批量创建任务 | 行内创建，Enter 连续 |

### 设计模式要素

1. **📋 单屏全景** — 所有待操作项在一个屏幕内可见
2. **⌨️ 键盘流** — Tab/Enter 在项之间无缝导航
3. **📊 进度指示** — 实时显示完成进度（如 "5/8 已填写"）
4. **🔘 批量动作** — 一键操作所有项（填充、提交、清空）
5. **💾 自动保存** — 防止意外丢失

## 8. 🎨 模板驱动设计

### 模板系统架构

模板系统让团队快速开始，无需从零配置：

| 功能模块 | 模板类型 | 预设选项 |
|---|---|---|
| 🗓️ 规划 | 迭代周期 | 1周 / 2周 / 3周 |
| 🔄 回顾 | 回顾框架 | Mad-Sad-Glad / 4Ls / Starfish / Start-Stop-Continue |
| 🤖 Agent | 人格风格 | 专业 / 友好 / 直接 |
| 🎨 外观 | 主题模式 | 浅色 / 深色 / 跟随系统 |

### 模板设计原则

- **🎯 一键应用** — 点击模板即生效，无需额外确认
- **🔄 随时切换** — 切换模板保留已有数据
- **⚡ 智能推荐** — 系统根据团队特征推荐默认模板
- **📋 透明展示** — 模板选项清晰标注适用场景

## 9. 📱 响应式设计策略

### 断点设计

| 断点 | 设备 | 布局策略 |
|---|---|---|
| > 1440px | 大屏桌面 | 完整双栏，Agent 面板展开 |
| 1024-1440px | 标准桌面 | 标准布局，Agent 面板可折叠 |
| 768-1024px | 平板 | 单栏为主，Agent 面板隐藏 |
| < 768px | 手机 | 纵向堆叠，横向滚动 |

### 看板响应式策略

- **📱 手机**: 看板列水平滚动，每列最小宽度 280px
- **📊 平板**: 双列显示，自适应列宽
- **🖥️ 桌面**: 四列并排，充分利用屏幕

### 核心原则

**「功能不减，布局自适应」** — 移动端不减少功能，只调整布局。

## 10. ♿ 无障碍设计考量

### 设计标准

Sprint Agent 遵循 WCAG 2.1 AA 级标准：

| 维度 | 实现方式 |
|---|---|
| 🎨 色彩对比 | 所有文本对比度 >= 4.5:1 |
| ⌨️ 键盘导航 | 所有功能可通过键盘访问 |
| 🔤 语义 HTML | 使用正确的标签和 ARIA 属性 |
| 🔍 焦点管理 | 清晰的焦点指示器 |
| 📖 屏幕阅读器 | 所有交互元素有描述标签 |

### 色彩无障碍

- 🏷️ **优先级标签** 不仅用颜色区分，还有文字标签
- ⭐ **评分星星** 同时支持点击和键盘 1-5 键
- 📊 **统计数字** 不依赖颜色传递信息

## 12. 🧪 UX 测试方法论（Multi-Agent 框架）

Sprint Agent v2.0 采用 **Multi-Agent 框架** 进行 UX 测试，模拟不同角色的真实使用场景。

### 测试框架架构

```
┌─────────────────────────────────────────────────┐
│              Multi-Agent UX 测试框架              │
├─────────────────────────────────────────────────┤
│                                                 │
│  🎭 UX 体验官 (40%)  ──→ 零知识直觉性测试        │
│         │                                       │
│  🏆 Scrum 主管 (30%) ──→ 工作流效率测试          │
│         │                                       │
│  👨‍💻 开发人员 A (15%) ──→ 任务执行效率测试       │
│         │                                       │
│  👩‍💻 开发人员 B (15%) ──→ 任务执行效率测试       │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 角色定义与测试重点

#### 🎭 UX 体验官（权重 40%）

- **角色设定**: 零前置知识的新用户
- **测试目标**: 直觉性测试 — 不阅读文档能否完成操作
- **测试场景**: 
  - 首次登录后能否独立创建 Sprint
  - 能否不看帮助完成一次站会
  - 能否直觉性地找到设置选项
- **评分维度**: 界面直观性、操作可发现性、错误恢复能力

#### 🏆 Scrum 主管（权重 30%）

- **角色设定**: 有敏捷经验的目标用户
- **测试目标**: 工作流效率 — 完成典型敏捷工作流的速度
- **测试场景**:
  - 创建 Sprint → 添加任务 → 管理迭代（端到端）
  - 每日站会的完整操作流程
  - 迭代回顾的完整操作流程
- **评分维度**: 流程完整性、操作效率、功能满足度

#### 👨‍💻👩‍💻 开发人员 ×2（权重各 15%）

- **角色设定**: 日常使用者
- **测试目标**: 高频操作效率 — 日常使用中的操作速度
- **测试场景**:
  - 快速创建 10 个任务
  - 批量拖拽任务变更状态
  - 站会批量填写
- **评分维度**: 操作速度、快捷键使用、流畅度

### 评分标准

| 等级 | 分数 | 说明 |
|---|---|---|
| 🟢 优秀 | 90-100 | 无需思考即可完成，体验愉悦 |
| 🟡 良好 | 75-89 | 基本流畅，偶有小困惑 |
| 🟠 一般 | 60-74 | 能完成但需思考，有改进空间 |
| 🔴 需改进 | < 60 | 操作困难，需要重新设计 |
